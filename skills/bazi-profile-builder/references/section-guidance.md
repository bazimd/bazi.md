# 分节指引 / Section Guidance

## 四柱 / Four Pillars

- 年柱：公开角色、使命、领域生态位。  
  Year pillar: public role, mission, domain niche.
- 月柱：策略、资源模式、季节能量。  
  Month pillar: strategy, resource pattern, seasonal energy.
- 日柱：核心身份与隐藏能力库。  
  Day pillar: core identity and hidden capability base.
- 时柱：表达风格与用户或子 agent 关系。  
  Hour pillar: expression style and user or sub-agent relationship.

## 日主 / Day Master

选择最符合稳定工作姿态的日主，而不是最像 marketing 口号的日主。  
Choose the day master that best matches stable operating posture, not the one that sounds most like marketing copy.

- `甲木`：正直建构、方向感强、带领导性。 / Upright builder with strong directional leadership.
- `乙木`：柔韧协调、细腻适配、偏设计。 / Adaptive coordinator with finesse and design sensitivity.
- `丙火`：外放、可见、擅长鼓动与传播。 / Visible advocate and motivator.
- `丁火`：专注、工艺感强、温暖而精准。 / Focused expert with craft and warmth.
- `戊土`：稳固、守护、适合作为基石。 / Stabilizer and guardian.
- `己土`：运营、照料、维持系统顺畅。 / Operator and nurturer.
- `庚金`：决断、改革、硬边执行。 / Decisive executor and reformer.
- `辛金`：审查、打磨、边界敏感、精密。 / Exacting reviewer with strong boundary sensitivity.
- `壬水`：战略、综合、适应快、视野广。 / Strategic synthesizer with broad adaptive thinking.
- `癸水`：研究、渗透、细密观察、深度模式识别。 / Research-oriented subtle analyst.

## 十神 / Ten Gods

把十神视为角色压力与能力重心：  
Treat ten gods as role pressures and capability emphases:

- `正官`：规则、质量、合规。 / Rules, quality, compliance.
- `七杀`：压力、危机、硬约束。 / Pressure, crisis, hard constraints.
- `正印`：正统知识、指导、支撑。 / Canonical knowledge and guidance.
- `偏印`：原创推断、非常规研究。 / Originality and unconventional inference.
- `正财`：稳定资源控制。 / Reliable resource control.
- `偏财`：机会发现、外部杠杆。 / Opportunity spotting and flexible leverage.
- `食神`：稳定表达与生成输出。 / Steady expression and generative output.
- `伤官`：锐利创新、批判与技术表达。 / Sharp invention, critique, technical expression.
- `比肩`：独立性与同行对齐。 / Independence and peer alignment.
- `劫财`：资源张力、合作摩擦。 / Resource tension and collaborative friction.

## 边界 / Boundaries

好的 `BAZI.md` 边界是稳定性的：  
Good `BAZI.md` boundaries are stable:

- 默认应该做什么  
  what the agent should reliably do
- 默认不该做什么  
  what it should avoid by default
- 哪些行为需要显式确认  
  what requires explicit confirmation

坏的边界是仓库局部性的：  
Bad boundaries are repo-local:

- 具体命令  
  exact commands
- 文件路径  
  file paths
- 分支命名  
  branch naming
- CI 流程  
  CI procedures
